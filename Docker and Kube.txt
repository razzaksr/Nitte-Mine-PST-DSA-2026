Build Maven CRUD app with SQL DAO Connection
Make sure Maven has
1. MySQL Dependency
2. Shaded build as follows to group MySQL within CRUD app jar
	<build>
  <plugins>
    <plugin>
      <groupId>org.apache.maven.plugins</groupId>
      <artifactId>maven-shade-plugin</artifactId>
      <version>3.5.0</version>
      <executions>
        <execution>
          <phase>package</phase>
          <goals>
            <goal>shade</goal>
          </goals>
          <configuration>
            <transformers>
              <transformer implementation="org.apache.maven.plugins.shade.resource.ManifestResourceTransformer">
                <mainClass>doc.kub.cli.App</mainClass>
              </transformer>
            </transformers>
          </configuration>
        </execution>
      </executions>
    </plugin>
  </plugins>
  </build>

3. build jar by "mvn clean package"
4. run the jar to verify all are working as expected


Containerization:
1. Turn the following windows feature ON
	a. Hyperviser
	b. Window for Linux system
2. Install docker desktop
3. Login into docker 

before proceed further docker cannot able to locate, localhost of MySQL server in your machine that is db url might be, 
jdbc:mysql://localhost:3306/test, which docker cannot connect, so change as follows "jdbc:mysql://host.docker.internal:3306/test"
do not need to change username and password.

4. create dockerfile in the root of maven project as following content in it
FROM eclipse-temurin:17-jdk

WORKDIR /app

COPY target/patient-cli-1.0-SNAPSHOT.jar app.jar

ENTRYPOINT ["java","-jar","app.jar"]

5. create docker image by
docker build -t patient-cli:v1 .

6. since this project is console we need to run container as 
docker run -it patient-cli:v1
where, 
-i = interactive input
-t = terminal

7. Tagging docker image
docker login
docker tag patient-cli:v1 razzaksr/patient-cli:v1
verify by "docker images" command

8. Push it to hub
docker push razzaksr/patient-cli:v1


Kubernetes:
1. install minikube.exe
2. run mimkube by 
minikube start --derver docker
3. create mysql-deployment.yaml
4. deploy mysql-deployment.yaml
5. create patient-cli-deployment.yaml
6. deploy patient-cli-deployment.yaml
7. can't run pod of console application
